Caps_Lock key 重绑定

大概是在两年前，刷知乎的时候看到用手掌外侧按 Ctrl 键的方法，当时刚学 Linux，此 方法在 Linux 下简直就是神技，因为在 Shell 中默认使用了 Emacs 的模式进行命令的编 辑，例如：Ctrl-a 将光标移动到行首，Ctrl-e 将光标移动到行末，Ctrl-b 将光标往左移 动一个字母等等。甚至在 Vim 里面也经常会用到 Ctrl 的组合键，例如：Ctrl-[ 退出插 入模式，其作用相当于 Esc 键。使用 Ctrl 的组合键的好处除了其本身代表的功能外，还 有重要的一点是提高打字的效率，我们的双手可以始终保持在键盘中心区域，而不需要为 了按个 Esc 键还得抬一下手。

这么好的技能，其实也有一些缺点。最近我也是饱受苦恼，因为左 Ctrl 键处于键盘的左 下角边缘，长时间使用手掌外侧按压，其方正的棱角会导致我的手掌渐渐产生疼痛感。

那有没有什么好的办法能解决这个问题呢？这一点的话 Emacs 用户应该是比较有心得的（ 因为他们经常要按 Ctrl 键，呵呵）。其实关于这个问题我早就了解过，Emacs 用户一般 都是将 Caps_Lock 键与 Ctrl 键互换，毕竟小拇指按 Caps_Lock 还是比较舒服的。 Richard Stallman 用的就是 HHKB 这种 Ctrl 在 Caps_Lock 位置的键盘。

当然我肯定不会为了这个而买一把 2000 多人民币的 HHKB。在 Linux 下有个 xmodmap 命 令，可以修改键位。于是花了点时间了解了下这条命令，发现其语法规则还是挺简单的， 新建一个 ~/.Xmodpad 文件，在里面写入如下内容： 

    clear Lock
    keysym Caps_Lock = Control_L
    add Control = Control_L

该命令表示取消 Caps_Lock 键原有的功能，将其重新绑定到 Control 键。之后执行 xmodmap ~/.Xmodpad 命令，键位即修改成功。不过，这样以后，我就得适应 Caps_Lock 的按键方法。今天尝试了一天，经常会和 Shift 键搞混在一起，看来得适应一段时间了， 呵呵。

Edit (2018/12/17):

在知乎看到了 这篇回答 ，发现还真 是个好办法，于是这两天又用 xmodmap 重新将键位映射了一遍，现在被我修改的键位如下 ：

  bind to
Caps_Lock Control_L
Alt_R BackSpace
Menu blackslash
Super_R Escape
